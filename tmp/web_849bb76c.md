# 频谱多标签分类项目方案
## （基于单标签版本的最小改动设计）

---

## 一、任务定义与目标

在原单标签分类任务（判断目标频谱为基频、低压倍频、高压倍频、组合频中的**唯一**一类）的基础上，扩展为**多标签分类**：目标频谱可能同时包含多个类别成分。

- **输入**：  
  一个时频谱图（目标频谱）和四个固定的参考掩码（基频、低压倍频、高压倍频、组合频）。  
  形状：`[F, T, 5]`（频率 × 时间 × 5通道）
- **输出**：  
  长度为4的多热编码 `[y1, y2, y3, y4]`，`yi ∈ {0,1}`，表示第i类是否存在。  
- **物理场景**：  
  实际转子振动信号中，多个频率族（基频、倍频、组合频）可能同时出现，网络需要独立识别每一种。

## 二、数据生成规范（MATLAB 端改动）

### 2.1 与单标签版本的差异

| 阶段 | 单标签版本 | 多标签版本（当前方案） |
|------|------------|------------------------|
| 样本构造 | 每一切片生成4个互斥样本，每个样本的目标频谱 = 第i类标签，标签为独热编码 | 每一切片生成 **1** 个样本，目标频谱 = 随机选择1~4类叠加，标签为多热编码 |
| 样本总数 | N_slices × 4 | N_slices（减少75%样本量，但可保留拼接扩增能力） |
| 标签形式 | `one‑hot [4]` | `multi‑hot [4]` |
| 输入X | `[F, T, 5, N]` | `[F, T, 5, N]`（通道定义不变） |
| 输出Y | `[N, 4]`独热 | `[N, 4]`多热 |

### 2.2 多标签样本生成步骤（单一切片）

设已由MATLAB函数 `generate_physics_layers` 获得当前切片的标签张量 `Labels` 和掩码张量 `Masks`，形状均为 `[T_slice, F, 4]`（4个类别的时频图）。

1. **随机抽取激活类别**：  
   - 类别编号集合 `{1,2,3,4}`。  
   - 随机决定子集大小 `k ∈ {1,2,3,4}`，然后从4类中无放回抽取 `k` 个类别。  
   - 记录多热标签 `y = [I(1∈激活), I(2∈激活), I(3∈激活), I(4∈激活)]`。

2. **合成目标频谱**：  
   - 将激活类别对应的标签频谱相加：  
     `target = sum( Labels(:,:,active_idx), 3 )`。  
   - 若需保证幅值范围合理，可归一化（但非必需）。

3. **组装5通道输入**：  
   - 通道1：`target`（形状 `[T_slice, F]`）  
   - 通道2：`Masks(:,:,1)`（基频掩码）  
   - 通道3：`Masks(:,:,2)`（低压倍频掩码）  
   - 通道4：`Masks(:,:,3)`（高压倍频掩码）  
   - 通道5：`Masks(:,:,4)`（组合频掩码）

4. **维度调整**：  
   - 最终样本输入形状为 `[F, T_slice, 5]`（频率在前，时间居中），存储于细胞数组，最后沿第4维拼接得到 `X = [F, T_slice, 5, N]`。  
   - 标签 `Y = [N, 4]` 多热矩阵。

5. **掩码保持不变**：掩码始终是四类频率的理论位置，不受激活类别影响，保证网络总能将目标与每个参考位置进行独立比对。

### 2.3 数据存储

- 保存为 `SpectrumClassificationData_multilabel.mat`，包含 `X`（四维）和 `Y`（二维）。  
- 同样在 Python 预处理阶段转换为 `(N,5,F,T)` 的 `.npy` 格式，以便复用原数据加载代码。

## 三、模型设计：多标签 ComparisonNet‑2D

### 3.1 复用原则

直接复用单标签版本的 **Comparator 子网络**，仅在顶层预测方式上做如下改动：

| 组件 | 单标签版本 | 多标签版本 |
|------|-----------|------------|
| Comparator 结构 | 同左 | 完全一致，不修改任何权重 |
| 并行调用方式 | 4次共享权重并行 | 完全一致 |
| 最终激活 | **Softmax**（输出归一化为四选一的概率） | **独立 Sigmoid** 或 **使用原始分数搭配 BCEWithLogitsLoss** |
| 输出含义 | 4个概率，和为1 | 4个独立概率，各自表示对应类别存在的置信度 |
| 损失函数 | CrossEntropyLoss（与独热标签配合） | **BCEWithLogitsLoss**（与多热标签配合） |

### 3.2 网络前向流程（同一批次 B）

| 步骤 | 操作 | 输入形状 | 输出形状 |
|------|------|----------|----------|
| 1. 拆分输入 | 取目标频谱 `target`（第0通道）与掩码 `masks`（第1~4通道） | `x_batch: [B,5,F,T]` | `target: [B,1,F,T]` <br> `masks: [B,4,F,T]` |
| 2. 逐类比较 | 对 i=1..4：<br> `mask_i = masks[:, i-1:i, :, :]` <br> `cmp_input = cat([target, mask_i], dim=1) → [B,2,F,T]` <br> `score_i = Comparator(cmp_input) → [B,1]` | | 4个 `[B,1]` 张量 |
| 3. 堆叠分数 | 将4个分数在特征维拼接 | `scores: [B,4]` | |
| 4. 损失计算 | 直接输入 `scores` 与多热标签 `y` 到 `BCEWithLogitsLoss` | `scores: [B,4]` <br> `y: [B,4]` | 标量损失 |
| 5. 推理后处理 | `prob = torch.sigmoid(scores)`，`pred = prob > threshold` (如0.5) | `scores: [B,4]` | `pred: [B,4]` 布尔型 |

### 3.3 设计理由

- **最小改动**：核心特征抽取器（Comparator）无需重训或调整，仅更换输出层逻辑和损失函数即可从单标签切换为多标签。  
- **物理一致性**：每个 Comparator 仍学习“目标频谱片段与某类掩码的匹配度”，多标签分类可视为4个独立的二分类“匹配检测器”。  
- **参数极简**：参数量不变（数百个），极难过拟合，特别适合小样本科学场景。  
- **无损对比**：可同时训练单标签与多标签版本，共享完全相同的 Comparator 结构，对比公平。

## 四、训练与评估流程

### 4.1 训练循环（单标签复用）

训练脚本 `train.py` 只需通过 `--task multilabel` 选择不同模型定义及损失函数，其余（数据加载、优化器、日志、检查点保存）完全复用。

- **损失函数**：`BCEWithLogitsLoss()`，内部执行 Sigmoid + 二值交叉熵，数值稳定。
- **监测指标**：除损失外，增加每个类别的 Binary Accuracy 和 Hamming Loss（见4.2）。
- **最佳模型选择**：根据验证集 Hamming Loss 或 Micro F1 决定。

### 4.2 评估指标（新增）

在 `utils/metrics.py` 中增加多标签专用指标函数，训练过程实时计算并写入 TensorBoard。

| 指标 | 计算方式 | 用途 |
|------|----------|------|
| **Hamming Loss** | 所有预测位中错误的比例，范围 [0,1]，越小越好 | 主要监测指标 |
| **Exact Match Ratio (子集准确率)** | 所有标签完全正确的样本比例 | 严格评估 |
| **Micro F1 / Macro F1** | 综合考虑精确率与召回率，微平均/宏平均 | 综合评价 |
| Per‑class F1 & AUC | 单独查看每一类的分类效果 | 诊断误判类别 |

建议在测试集中汇总一份对比表格（单标签模型 vs 多标签模型）展示上述指标。

### 4.3 训练数据混合策略（可选提升）

- 为增强模型对单标签场景的区分能力，可在训练时混合部分“单激活”样本（即 `k=1` 的样本）。  
- 也可以动态生成样本，每次迭代随机组合激活类别，实现数据扩增。

## 六、实施步骤（与单标签对比并行）

1. **数据生成**  
   - 修改 MATLAB 脚本，实现2.2节所述多标签样本构造方法，生成 `_multilabel.mat`。  
   - 运行现有 `preprocess.py` 转换为 `.npy`。

2. **实现多标签网络**  
   - 在 `models/multilabel_comparison_net.py` 中构建 `MultilabelComparisonNet`，内部调用与单标签相同的 `Comparator` 类（从 `comparison_net_2d` 导入），前向时输出 `scores`（不做 Sigmoid）。  
   - 无需重复编写 Comparator。

3. **更新训练入口**  
   - 在 `train.py` 中添加分支：当选择多标签模型时，实例化 `MultilabelComparisonNet`，损失函数设为 `nn.BCEWithLogitsLoss()`，Y 直接从多热矩阵读取。

4. **调整评估流程**  
   - 在 `utils/metrics.py` 中实现多标签指标，训练时每 epoch 打印 Hamming Loss 和 Micro F1，并写入 TensorBoard。

5. **对比实验**  
   - 使用相同数据划分（仅标签从独热变为多热），训练单标签 baseline 和多标签网络，记录：  
     - 验证集多指标（单标签网络预测可通过取 `argmax` 转为单标签，再与多热标签比较，但意义不同；建议直接对比多标签指标）。  
     - 参数量、推理速度。  
   - 将结果整理成对比表格，分析多标签建模的优势与适用场景。

6. **可视化与展示**  
   - 随机抽检样本，绘制目标频谱、四个掩码、四个概率值并排图，直观验证网络匹配学习效果。  
   - 绘制 Hamming Loss / F1 随 epoch 变化曲线。

## 七、总结

本方案以单标签项目为基础，通过**最少改动**实现多标签频谱分类：  
- 数据生成端仅改变每切片的合成策略，掩码保持不变。  
- 模型复用同一 Comparator，仅替换输出层和损失函数。  
- 训练、评估、日志等基础设施可直接继承，大幅降低实现与验证成本。  

这种设计不仅使单标签与多标签模型具有完全可比的网络容量和特征抽取能力，也便于在后续研究中统一框架、快速实验，为博士学位论文中频谱分析相关章节提供扎实的对比实验支撑。

## 三、缓冲策略与风险应对

- **Day 1-2**：若数据生成遇到维度不匹配或随机组合逻辑错误，可在 Day 2 继续修正，不影响后续。
- **Day 3-4**：若模型前向或训练入口报错，有充足时间调试，甚至可重读 `Comparator` 接口。
- **Day 5**：全量训练若因超参数不理想（如学习率过大导致不收敛），还有时间重新调整后再次训练。

建议在 Day 4 完成小样本训练后，确定学习率、batch size 等微调方向，Day 5 一次性完成正式训练。
